# Non-Turn Signage System — 技術判断・仕様確定事項

確定日: 2026-03-17

---

## A. アーキテクチャ

### A1. データ永続化: Turso (libSQL)
- https://turso.tech/ — エッジ対応の軽量SQLiteベースDB
- MCP連携あり
- 1端末あたり10〜50枚のプレイリスト規模
- 複数端末展開を見据えリレーショナルDBが最適

### A2. 認証: なし（MVP）
- MVP段階では認証を実装しない
- 将来的に必要になった時点で追加する

### A3. 複数端末対応
- **MVP: 単一端末**の動作を優先
- **DB設計のみ**: 最初から `device_id`, `store_id` カラムを持たせる
- 系列店拡大時に根本的な作り直しを回避

---

## B. 表示仕様

### B1. ディスプレイ
- **解像度**: 4K (3840x2160) ターゲット
- **向き**: portrait（縦）+ landscape（横）切り替え対応
- 4Kモニターでもピクセルがぼやけない設計

### B2. 画像仕様
- **対応フォーマット**: JPEG, PNG, WebP, PDF（1ページ=1スライドとして展開表示）
- **最大ファイルサイズ**: 30MB/枚
- **リサイズ・最適化**: R2アップロード時には**行わない**
- 理由: プロの商業カメラマンのレタッチ済み写真のディテール（暗部の階調・色味）を保持
- オフライン同期型アーキテクチャなので、重いファイルでも再生時に遅延なし

## D. 開発優先度

### D1. 着手順序
1. **Local Viewer（FE）** — コアバリューの証明が最優先
   - パターンCのぼかし背景、4K対応のフェード表示
   - ダミー高画質画像で検証
2. **Cloud Admin API（BE）** — Viewer動作後に着手
3. **Sync Agent（BE）** — API完成後に着手
4. **Cloud Admin UI（FE）** — API完成後に並列着手可能
5. **インフラ（PM2, キオスクモード）** — 最終フェーズ

### D2. Cloud Admin UI品質
- **Apple風デザイン（apple-design スキル適用）**
- Glassmorphism活用、滑らかなアニメーション
- クリエイターが作品を管理するツールとして、余白を活かしたミニマルで美しいUI

## コアバリュー（設計の指針）
> 「重い画像を、構図を崩さず（パターンCのぼかし背景）、カクつくことなく美しくフェードさせること」

このプロジェクトの存在意義。すべての技術判断はこの価値を守るために行う。
